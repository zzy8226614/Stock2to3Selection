# 二进三选股系统 Spec

## 0. 版本记录

| 版本 | 日期 | 变更内容 |
| :--- | :--- | :--- |
| `v0.1` | `2026-06-12` | 根据《二进三选股需求书》建立首版产品、接口、算法与多端实现规范 |

## 1. 项目目标

本项目基于 `MoneyAPP` 的成熟工程能力裁剪改造，形成独立的二进三选股系统。首版保留两个既有能力：

1. `情绪信号`
2. `板块个股排名`

首版新增核心能力：

1. `二板解析`

后端目标部署到阿里云：

| 项 | 值 |
| :--- | :--- |
| 公网 IP | `47.107.125.248` |
| 服务端口 | `8080` |
| 协议 | `HTTP` |
| 建议基础地址 | `http://47.107.125.248:8080/` |

Android App 与 PC 桌面应用均需同步改造，默认入口只展示首版保留和新增的三个功能。

## 2. 功能范围

### 2.1 情绪信号

保留 MoneyAPP 已实现功能。用于判断当日市场状态，输出：

| 字段 | 说明 |
| :--- | :--- |
| `trade_date` | 交易日 |
| `marketOverview` | 大盘表现 |
| `turnoverOverview` | 成交额 |
| `regimeLabel` | 情绪判定：红灯 / 黄灯 / 绿灯 |
| `positionAdvice` | 仓位建议 |
| `indicators` | 涨停家数、最高连板、连板晋级率、跌停潮 |

### 2.2 板块个股排名

保留 MoneyAPP 已实现功能。穷举板块排名前 10 的全部涨停个股，并按板块排名、个股质量排序展示。

### 2.3 二板解析

新增按钮与接口。功能目标：

1. 穷举当日所有二板股票。
2. 排除创业板、科创板、北交所、ST 股。
3. 按所属板块排名依次展示。
4. 输出需求书指定字段。
5. 新增 `首板量能`，用于衡量今日二板成交额相对首板前 5 个交易日平均成交额的倍数。

## 3. 二板解析筛选口径

### 3.1 入池条件

| 条件 | 规则 |
| :--- | :--- |
| 连板数 | `连板数 == 2` |
| 板块 | 必须能识别所属行业或板块，否则归为 `未知板块` |
| 股票名称 | 排除包含 `ST` 的股票 |
| 股票代码 | 排除 `300` 创业板、`688` 科创板、`8`/`4` 北交所 |
| 数据日期 | 未来日期或当日未收盘时不回填历史缓存，返回空结果和说明 |

### 3.2 排序规则

排序优先级固定如下：

1. `boardRank` 升序，板块排名越靠前越先展示。
2. `boardLimitUpCount` 降序，同板块涨停数越多越先展示。
3. `firstBoardEnergy` 降序，可计算首板量能的样本优先。
4. `sealTime` 升序，封单越早越先展示。
5. `symbol` 升序，保证结果稳定。

### 3.3 首板量能

`首板量能`定义：

```text
首板量能 = 今日成交额 / 首板前 5 个交易日平均成交额
```

处理规则：

| 场景 | 展示值 |
| :--- | :--- |
| 可计算且倍数 >= 1 | 保留两位小数，例如 `1.35` |
| 可计算但倍数 < 1 | `0` |
| 历史数据缺失、成交额字段缺失、平均成交额为 0 | `W` |

说明：当上游数据只提供日线历史时，系统使用二板当日成交额近似“首板交易额”，避免额外分钟级依赖导致接口不可用。若后续接入更精确首板封板时点成交额，可在不破坏字段的前提下升级算法说明。

## 4. 展示字段

二板解析单票展示格式：

| 字段 | 接口字段 | 说明 |
| :--- | :--- | :--- |
| 股票名称 | `stockName` | 股票名称 |
| 股价 | `latestPrice` | 最新价，单位元 |
| 流通市值 | `floatMarketCap` | 格式化为亿元 |
| 所属板块 | `boardName` | 行业/板块名称 |
| 板块排名 | `boardRank` | 以当日板块涨停数排名 |
| 板块涨停数 | `boardLimitUpCount` | 所属板块涨停个股数量 |
| 封单时间 | `sealTime` | 首次封板时间 |
| 封单手数 | `sealOrderLots` | 封板资金 / 最新价 / 100 |
| 开板次数 | `openBoardCount` | 炸板次数 |
| 换手率 | `turnoverRate` | 当日换手率 |
| 首板量能 | `firstBoardEnergy` | 见第 3.3 节 |

## 5. API 契约

### 5.1 Legacy 接口

| 方法 | 路径 | 用途 |
| :--- | :--- | :--- |
| `GET` | `/health` | 健康检查 |
| `POST` | `/screen/market-signal` | 情绪信号 |
| `POST` | `/screen/board-top10-limit-up` | 板块个股排名 |
| `POST` | `/screen/second-board-analysis` | 二板解析 |

请求体：

```json
{
  "trade_date": "2026-06-12",
  "use_demo_on_failure": true,
  "force_refresh": false
}
```

### 5.2 `/api/v1` 接口

新客户端优先使用 `/api/v1`：

| 方法 | 路径 |
| :--- | :--- |
| `GET` | `/api/v1/health` |
| `POST` | `/api/v1/screen/market-signal` |
| `POST` | `/api/v1/screen/board-top10-limit-up` |
| `POST` | `/api/v1/screen/second-board-analysis` |

`/api/v1` 统一返回 envelope：

```json
{
  "success": true,
  "data": {},
  "meta": {
    "requestId": "uuid",
    "clientType": "android",
    "cacheHit": false,
    "degraded": false,
    "source": "live",
    "upstreamSource": "live"
  },
  "error": null
}
```

## 6. 客户端改造

### 6.1 Android

首页按钮只保留：

1. `情绪信号`
2. `二板解析`
3. `板块个股排名`

默认后端地址改为：

```text
http://47.107.125.248:8080/
```

二板解析结果页使用两行紧凑字段布局：

1. 股价、流通市值、所属板块、板块排名、板块涨停数
2. 封单时间、封单手数、开板次数、换手率、首板量能

### 6.2 PC 桌面端

桌面端 Web 壳保留诊断信息、请求历史和刷新能力。首页功能卡片只保留：

1. `情绪信号`
2. `二板解析`
3. `板块个股排名`

## 7. 测试要求

后端至少覆盖：

1. `/screen/second-board-analysis` 基础响应结构。
2. `/api/v1/screen/second-board-analysis` envelope 响应结构。
3. 二板解析排除创业板、科创板、北交所、ST。
4. 首板量能可计算、低于 1、不可计算三类展示值。
5. 未来日期不回填历史缓存。

客户端至少验证：

1. 首页按钮与需求范围一致。
2. 二板解析结果字段完整。
3. 刷新按钮会携带 `force_refresh=true`。
4. 后端地址默认值为 `http://47.107.125.248:8080/`。

## 8. 非目标

首版不实现自动交易、买卖点提醒、回测、账户风控和盘中实时监控。现阶段系统定位为收盘后二进三候选分析工具。
